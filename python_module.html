1. ¿Qué es el Backend?

El **Backend** (o "lado del servidor") es la parte de una aplicación web que el usuario no ve. Es el motor que trabaja "detrás del escenario".

Se encarga de la lógica de negocio, de procesar la información, de conectarse a la **base de datos** para guardar y leer información, y de la seguridad (como la autenticación de usuarios). Python es increíblemente popular para esto gracias a su simplicidad y su potente ecosistema.

2. Sintaxis Básica de Python

La sintaxis de Python es famosa por ser limpia y legible. No usa llaves {} ni punto y coma al final de las líneas. En su lugar, usa la **indentación** (el espacio al inicio de la línea) para definir bloques de código. 


# Esto es un comentario en Python

# Declaración de variables (no necesitas 'let' o 'const')
nombre = "Code Master"
puntos = 100

# Definición de una función (fíjate en 'def' y la indentación)
def saludar(usuario):
  if usuario:
    print(f"Hola, {usuario}! Bienvenido al backend.")
  else:
    print("Hola, mundo!")

# Llamar a la función
saludar(nombre)

3. Frameworks (Flask)

Aunque puedes escribir un backend desde cero, es mucho más fácil y rápido usar un **framework**. Un framework te da herramientas y una estructura para construir aplicaciones robustas.

**Flask** es un "micro-framework" de Python, perfecto para empezar. Es simple y te permite crear una aplicación web en muy pocas líneas de código. 


# Un servidor web básico con Flask (ejemplo conceptual)
# (Necesitarías instalar Flask: 'pip install Flask')

from flask import Flask
app = Flask(__name__)

# Se define una "ruta". Esto se ejecuta cuando alguien visita "/"
@app.route("/")
def hola():
  # La función devuelve el HTML que verá el usuario
  return "<h1>Hola desde el servidor de Python!</h1>"

# Esto iniciaría el servidor (ejecutando 'python app.py')

4. ¿Qué es una API?

Una **API** (Application Programming Interface) es un conjunto de reglas que permite que diferentes aplicaciones se comuniquen entre sí.

En el backend, tú creas una API (generalmente una **API REST**) para que tu frontend (hecho con HTML/CSS/JS o React) pueda pedirle ("GET") o enviarle ("POST") datos en un formato universal como **JSON**. El backend recibe la petición, va a la base de datos y le responde al frontend solo con los datos que necesita.
